Hva er semantisk HTML?

Semantisk HTML betyr å bruke elementer som beskriver innholdets rolle, f.eks. <header> for toppseksjon, <nav> for navigasjon og <footer> for bunntekst.

Artikler

Bruk av <section> vs. <div>

<section> brukes for tematiske grupper med overskrift, mens <div> er nøytralt og brukes kun når ingen andre elementer passer.

Når passer <aside>?

<aside> brukes for innhold som er relatert, men ikke hoveddelen, som tips eller ekstra informasjon.

Forklaring på semantisk HTML

Semantisk HTML betyr at man bruker HTML-elementer som beskriver meningen med innholdet, i stedet for bare utseendet. Eksempler er <header>, <nav>, <main>, <article> og <footer>.

Fordeler

  • Bedre tilgjengelighet for skjermlesere
  • Bedre SEO (søkemotoroptimalisering)
  • Mer lesbar og forutsigbar kode
  • Standardisert struktur

Ulemper

  • Litt læringskurve
  • Kan misbrukes hvis feil elementer velges
  • Overkill for svært enkle sider

Alternativet: Ikke-semantisk HTML

Det motsatte er å bruke generelle elementer som <div> og <span> til nesten alt. Fordelen er at det er raskt og enkelt, men ulempen er at koden blir mindre tilgjengelig og dårligere forstått av søkemotorer.
